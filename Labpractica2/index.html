Practica 2: Configuración de Red Local

Protocolos Arduino

- Introducción -

Antes de adentrarnos en la práctica de configuración de una red local, es esencial comprender algunos conceptos básicos que serán fundamentales para el desarrollo y entendimiento de esta actividad.
En el ámbito de la informática y redes, es esencial comprender y dominar diversos aspectos relacionados con la conectividad, configuración de servidores locales y la administración de redes. Este proceso abarca desde la conexión a redes inalámbricas hasta la configuración de servicios esenciales como Apache a través de herramientas como Xampp. En este contexto, se describirán los pasos para cumplir con una serie de objetivos específicos, diseñados para proporcionar una comprensión profunda de las habilidades necesarias en un entorno de redes universitario. Una red local (LAN) es un conjunto de equipos informáticos y dispositivos conectados entre sí en un área geográfica relativamente pequeña, como puede ser un edificio. Estas redes permiten compartir recursos y datos de manera eficiente y segura.
La dirección IP es un número único asignado a cada dispositivo conectado a una red que utiliza el Protocolo de Internet para la comunicación. Este identificador permite que los dispositivos se localicen y comuniquen entre sí en una red.
XAMPP es un paquete de software gratuito que incluye el servidor web Apache, el gestor de bases de datos MariaDB, y los lenguajes de programación PHP y Perl. XAMPP facilita la creación de un entorno de servidor web local para el desarrollo y prueba de páginas y aplicaciones web.
El firewall o cortafuegos es un sistema de seguridad que controla el tráfico entrante y saliente en una red, basándose en un conjunto de reglas predefinidas, con el objetivo de proteger los dispositivos de accesos no autorizados.
Con estos conceptos claros, procederemos a detallar el desarrollo de la práctica de configuración de una red local, siguiendo los objetivos propuestos.

- Objetivo General -

Conocer, investigar e identificar los pasos para configurar una red local y el acceso a recursos de un equipo en red.

- Objetivos Especificos -

  • Conecta tu computadora a la red de la universidad mediante la conexión wifi.
  • Identifica la Ip de acceso a la red de tu computadora.
  • En tu computadora instala Xampp para configurar un servidor local, habilita los servicios de Apache y monta un sitio web: https://www.apachefriends.org/es/index.html
  • Habilita el acceso al puerto 80 desde el firewall del equipo configurado como el servidor local.
  • Comprueba el acceso al sitio web montado en el servidor local desde 3 dispositivos distintos (para acceder es necesario estar en la misma red).
  • Elabora un mapa de accesos de equipos y sus respectivas IPs en la Red de las computadoras de tus demás compañeros.
  • Repite los mismos pasos ahora utilizando el cable directo y conectándote a la red de la universidad cableada.
  • Repite los mismos pasos, pero ahora con el cable cruzado entre dos equipos, configurando la IP de ambos equipos a 192.168.0.1 (un equipo) y el 192.168.0.2 (el segundo equipo).

- Desarrollo -

Desarrollo en base a los objetivos especificos

Conexión a la Red de la Universidad Mediante la Conexión WiFi

Identificación de la IP de Acceso a la Red de la Computadora
Una vez establecida la conexión a la red, se procedió a identificar la dirección IP de la computadora. Este paso fue crucial para determinar la asignación de la red y asegurar la correcta comunicación con otros dispositivos. La dirección IP fue identificada utilizando herramientas de línea de comandos disponibles en el sistema operativo, lo que permitió obtener detalles adicionales sobre la configuración de red del equipo, incluyendo la máscara de subred y la puerta de enlace.

Instalación de XAMPP y Configuración de un Servidor Local
La instalación de XAMPP representó un momento clave en la práctica, ya que este software permitió emular un servidor web en la computadora. Tras la instalación, se iniciaron los servicios de Apache, poniendo en marcha el servidor local. A continuación, se procedió a montar un sitio web de prueba, accesible inicialmente solo desde la computadora servidor a través de la dirección `http://localhost`. Esta configuración sirvió como base para las pruebas de conectividad y acceso desde otros dispositivos en la red.

Habilitación del Acceso al Puerto 80 Desde el Firewall
Para que otros dispositivos en la red pudieran acceder al sitio web montado en el servidor local, fue necesario configurar el firewall de la computadora para permitir el tráfico a través del puerto 80, el cual es utilizado por defecto por el servidor web Apache. Este ajuste implicó modificar las reglas del firewall para aceptar conexiones entrantes, garantizando así que las peticiones externas al servidor web fueran correctamente recibidas y procesadas.

Comprobación del Acceso al Sitio Web Desde Distintos Dispositivos
Con el servidor web en funcionamiento y el firewall correctamente configurado, se procedió a verificar el acceso al sitio web de prueba desde tres dispositivos diferentes conectados a la misma red WiFi de la universidad. Este paso confirmó la exitosa configuración del servidor local y la accesibilidad del sitio web en la red local, demostrando la funcionalidad del entorno de desarrollo configurado.

Elaboración de un Mapa de Accesos de Equipos en la Red
Utilizando herramientas de escaneo de red, se elaboró un mapa que reflejaba los dispositivos conectados a la red local y sus respectivas direcciones IP. Este mapa proporcionó una visión clara de la estructura de la red de la universidad, identificando no solo las computadoras de los compañeros, sino también otros dispositivos conectados, lo cual facilitó la comprensión de la dinámica de la red y la identificación de posibles recursos compartidos.

Configuración de Conexión Directa y con Cable Cruzado
Finalmente, se experimentó con la configuración de la red utilizando un cable directo y luego un cable cruzado, conectando dos equipos directamente entre sí. Este proceso implicó la asignación manual de direcciones IP estáticas (192.168.0.1 y 192.168.0.2) para cada equipo, permitiendo la comunicación directa sin necesidad de un dispositivo de red intermedio como un router o switch. Este ejercicio práctico destacó las diferencias en la configuración de la red y proporcionó una valiosa experiencia en la configuración de conexiones de red punto a punto.
Cada uno de estos objetivos específicos contribuyó a la comprensión y aplicación práctica de los principios fundamentales de configuración y gestión de redes locales, proporcionando una base sólida para futuros aprendizajes y exploraciones en el campo de las redes informáticas.
